REASONABLE-ACCOMMODATION-INTAKE-READY

Accommodation requests are not ordinary support tickets. A fast AI-assisted reply can accidentally ask for the wrong proof, disclose private health or disability details, deny a request, promise approval, quote a policy incorrectly, or route the issue to the wrong person.

This intake card keeps the first response neutral and useful while preserving human review for legal, fair-housing, lease, privacy, accessibility, medical, fee, parking, animal, entry, or policy-exception decisions.

Copy/paste accommodation request intake card

Resident / applicant / requester: [name, unit or prospect record, preferred contact]
Property / unit / application: [property, unit, application ID, lease status]
Request category: [accessibility / assistance animal / parking / communication method / entry/access / deadline / policy exception / other]
Requested change or help: [resident's words; do not paraphrase into a legal conclusion]
Date received and channel: [date/time + email/portal/phone/SMS/in-person]
Source records to review: [lease, house rules, application notes, prior accommodation records, approved policy source, legal/manager contact]
Privacy-sensitive details present: [medical details / disability information / assistance animal details / private household information / none known]
Immediate safety or habitability concern: [yes/no + details]
Resident-safe acknowledgement: [short neutral acknowledgement]
Reviewer / owner: [property manager / compliance contact / legal / owner / regional manager]
CRM label: ACCOMMODATION-INTAKE / PRIVACY-SENSITIVE / NEEDS-MANAGER-REVIEW / NEEDS-LEGAL-ROUTE / ACCESSIBILITY-REQUEST
Next safe action: [acknowledge receipt / route to reviewer / request approved form or contact path / confirm timeline source / hold automation]
Do-not-say until reviewed: [approved/denied, legal conclusion, medical judgement, fee waiver, lease interpretation, exception promise, enforcement threat]

Resident-safe first replies

1. Request received

Thanks for reaching out. We received your request and are routing it through the property's review process so the right person can respond using the current records and approved contact path.

2. Need approved contact path

Thank you for the details. To make sure this is reviewed correctly, please use [approved accommodation/contact path] or send the requested information to [approved contact]. We will keep this note with your record.

3. Privacy-sensitive details present

We appreciate you sharing this. Because the request may include private or accommodation-sensitive information, we are routing it to the appropriate reviewer rather than handling it in a general message thread.

Review this draft accommodation-sensitive property-management reply.

Goal: acknowledge the request, preserve privacy, route to the approved reviewer, and avoid legal/fair-housing conclusions.

Verified facts only:
[Paste approved policy/contact path, request date, property record, and reviewer instructions]

Draft reply:
[Paste draft]

Check for:
1. Any approval, denial, medical judgement, disability assumption, legal conclusion, lease interpretation, fee promise, parking promise, animal-policy conclusion, or deadline invented without review.
2. Any unnecessary private health, disability, household, animal, applicant, or resident-identifying detail.
3. Any request for documentation that is not on the approved process/source.
4. Any tone that sounds punitive, dismissive, retaliatory, or inconsistent.
5. A safer version under 120 words that routes to the approved reviewer.

Stop rules before automation

  • STOP AUTOMATION before approving, denying, delaying, conditioning, charging, waiving, enforcing, or interpreting any accommodation, accessibility, assistance-animal, parking, entry, application, lease, fee, or policy-exception request.
  • Do not invent medical requirements, documentation rules, proof standards, deadlines, lease terms, legal status, fair-housing conclusions, owner decisions, or prior request history.
  • Keep private health, disability, assistance-animal, household, applicant, and resident details inside the approved property-management system.
  • Use manager/legal/compliance-reviewed follow-up for any outcome, deadline, denial, approval, or exception language.
