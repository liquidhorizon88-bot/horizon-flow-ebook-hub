AI Search Replacement Part Compatibility Checklist for Small Businesses

REPLACEMENT-PART-COMPATIBILITY-SOURCE-CONTROL-READY — Use this worksheet before AI search, chatbots, support macros, product pages, marketplace listings, or staff replies explain whether a replacement part fits a product.

Replacement-part compatibility answers are easy for AI tools to overstate. A single invented model number, substitute part, installation note, return promise, or warranty statement can turn a support shortcut into a customer-service problem. This checklist gives small businesses a source card, safe wording pattern, AI audit prompt, and stop rules for compatibility-sensitive answers.

1) Replacement-part compatibility source card

  • Product / model: {product_name}, {model_number}, {sku}, {serial_range_or_generation}
  • Replacement part: {part_name}, {part_number}, {revision}, {vendor_or_manufacturer}
  • Approved source links: product page, parts diagram, manual, compatibility matrix, vendor spec sheet, internal support note, return policy
  • Fit status: fits / does not fit / needs adapter / needs owner review / discontinued / superseded by {approved_part}
  • Proof required: model photo, serial number, purchase date, installation environment, firmware/software version, measurements, region/voltage/version notes
  • Review owner: support / product manager / technician / manufacturer / vendor, plus {review_date} and {next_review_date}

2) Customer-safe wording snippets

Confirmed fit: “Based on our active compatibility source, {part_number} is listed for {product_or_model}. Please verify {proof_needed} before ordering or installing.”

Needs verification: “I do not want to guess from a similar model. Please send {model_or_serial_proof}, and we will compare it against the active compatibility source before recommending a part.”

Superseded part: “The older part appears to have been replaced by {approved_substitute}, but compatibility still depends on {condition}. We will verify the active source before promising fit or availability.”

Not supported: “Our current source does not list that part as compatible with {model}. We can help identify the correct part if you share {proof_needed}.”

STOP PUBLISHING guardrails

  • Stop if AI output invents a model number, part number, fit status, substitute, installation step, return eligibility, warranty outcome, or availability claim not present in the approved source.
  • Stop if product pages, manuals, diagrams, vendor PDFs, and support macros disagree; assign an owner before publishing or sending the answer.
  • Do not promise fit from product names alone when serial range, generation, firmware, dimensions, voltage, region, or adapter requirements matter.
  • Do not tell customers a part is returnable, refundable, covered, safe to install, or equivalent unless the active policy/source supports that exact statement.
  • Keep discontinued and superseded parts separate from active compatible parts so AI search does not quote outdated instructions.

3) AI audit prompt

Audit this replacement-part compatibility answer before publishing or sending it.
Use only the approved source card below. Do not invent part numbers, fit status, substitutes, adapter requirements, installation steps, return eligibility, warranty coverage, availability, pricing, or safety claims.
If the sources conflict or compatibility depends on model/serial/generation proof, return "NEEDS OWNER REVIEW" and list the missing fields.
Approved source card:
- Product / model: {product_name}, {model_number}, {sku}, {serial_range}
- Replacement part: {part_name}, {part_number}, {revision}
- Active source URL/PDF: {active_source}
- Fit status: {approved_fit_status}
- Conditions/adapters/exclusions: {conditions}
- Return/warranty note: {approved_policy_note}

Draft answer to audit:
{draft_answer}

4) Mini source-review checklist

  • List every public page, manual, PDF, diagram, support macro, marketplace listing, and vendor note that mentions replacement parts or compatibility.
  • Mark one active compatibility source for each part family; annotate or remove stale sources that mention old model numbers.
  • Create safe customer wording for confirmed fit, unknown fit, not supported, discontinued, and superseded part scenarios.
  • Add proof fields to the support form so model/serial/generation evidence is captured before a recommendation is made.
  • Retest chatbot, FAQ, product-page, and marketplace answers after every part/vendor/manual update.
